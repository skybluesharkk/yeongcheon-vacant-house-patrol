# 통합 대시보드 조회

Category: Front → Backend
Content-Type: -
Endpoint: /api/dashboard
Main Fields: selectedRobotId(query)
Main Response: weather, robots[].latestImage, robots[].latestLidar.previewImageUrl, robots[].latestLidar.summary, robots[].patrolPath, vacantHouseMap, maintenancePriorities, reconstruction, urgentEvent, stats
Method: GET
Purpose: 대시보드 전체 데이터 조회

# 통합 대시보드 조회

## 핵심 수정

로봇이 여러 대이므로 `latestImage`, `latestLidar`, `patrolPath`를 최상위 단일 필드로 두면 안 된다.

대시보드 통합 조회에서는 로봇별로 이미지, 라이다, 경로를 따로 내려준다.

```
robots[].latestImage
robots[].latestLidar
robots[].patrolPath
```

---

## API

```
GET /api/dashboard
```

## Query Parameter

| 이름 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| --- | --- | ---: | --- |
| selectedRobotId | string | X | 프론트에서 기본 선택할 로봇 ID. 없으면 첫 번째 운행 로봇 기준 |

## 프론트 사용 방식

프론트는 `robots` 배열을 순회해서 로봇 목록과 지도 마커를 그리고, 선택된 로봇은 `selectedRobotId`로 찾는다.

```
selectedRobot = robots.find(robot => robot.robotId === selectedRobotId)
```

선택된 로봇의 카메라/라이다 화면은 아래처럼 접근한다.

```
selectedRobot.latestImage
selectedRobot.latestLidar
selectedRobot.patrolPath
```

---

## 별도 상세 API와의 관계

`GET /api/dashboard`는 전체 화면을 한 번에 그리기 위한 API다.

`GET /api/dashboard/robots/{robotId}`는 로봇 하나를 클릭했을 때 더 자세한 이미지/라이다/이벤트를 가져오는 API로 사용한다.
