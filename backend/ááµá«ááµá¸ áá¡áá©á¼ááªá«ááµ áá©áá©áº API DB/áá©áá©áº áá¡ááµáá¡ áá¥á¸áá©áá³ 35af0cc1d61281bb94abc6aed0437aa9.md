# 로봇 라이다 업로드

Category: Robot → Backend
Content-Type: multipart/form-data
Endpoint: /api/robots/{robotId}/lidar
Main Fields: lidarFile, x, y, timestamp
Main Response: success, robotId, lidarId, lidarFileUrl, previewImageUrl, summary
Method: POST
Purpose: 로봇별 라이다 NumPy 배열 업로드 후 백엔드에서 Preview/정량 summary 생성

# 로봇 라이다 업로드

## 핵심 수정

라이다 Preview 생성은 AI Agent가 담당하지 않는다.

라이다 `.npy` / `.npz`는 정량 센서 데이터이므로, 백엔드 또는 백엔드 내부 Python utility가 파일을 로드해서 Preview 이미지와 정량 summary를 생성한다.

```
Robot → Backend
.npy/.npz 업로드

Backend
NumPy 로드
좌표/거리 계산
Preview 이미지 생성
summary 계산
DB 저장

Frontend
GET /api/dashboard에서 previewImageUrl + summary 조회
```

---

## API

```
POST /api/robots/{robotId}/lidar
Content-Type: multipart/form-data
```

## Form Data

| 필드 | 타입 | 필수 | 예시 | 설명 |
| --- | --- | --- | --- | --- |
| --- | --- | ---: | --- | --- |
| lidarFile | file | O | `scan_001.npy` | `.npy` 또는 `.npz` 라이다 배열 |
| x | number | O | `12.4` | 측정 위치 x |
| y | number | O | `8.7` | 측정 위치 y |
| timestamp | string | O | `2026-05-20T14:35:22` | 측정 시각 |

---

## Backend 처리

| 순서 | 처리 |
| --- | --- |
| 1 | `.npy` 또는 `.npz` 파일 저장 |
| 2 | 백엔드 내부 Python utility 또는 parser로 NumPy 배열 로드 |
| 3 | point cloud 또는 2D scan 좌표 계산 |
| 4 | 대시보드 표시용 Preview 이미지 생성 |
| 5 | `pointCount`, `minDistance`, `maxDistance`, `obstacleCount` 등 정량 summary 계산 |
| 6 | `lidar_scan` 테이블에 파일 URL, Preview URL, summary 저장 |
| 7 | `GET /api/dashboard`의 `robots[].latestLidar`에 반영 |

---

## 저장 필드

| 필드 | 설명 |
| --- | --- |
| robot_id | 로봇 ID |
| lidar_file_url | 원본 `.npy/.npz` 파일 URL |
| preview_image_url | 백엔드가 생성한 Preview 이미지 URL |
| point_count | 포인트 개수 |
| min_distance | 최소 거리 |
| max_distance | 최대 거리 |
| avg_distance | 평균 거리 |
| obstacle_detected | 장애물 감지 여부 |
| obstacle_count | 장애물 수 |
| front_blocked | 전방 막힘 여부 |
| left_blocked | 좌측 막힘 여부 |
| right_blocked | 우측 막힘 여부 |
| x | 측정 위치 x |
| y | 측정 위치 y |
| created_at | 생성 시각 |

---

## 비고

`/agent/lidar/render-preview`는 제거 대상이다.

라이다 Preview는 AI Agent가 아니라 백엔드의 데이터 처리 로직으로 보는 것이 맞다.
